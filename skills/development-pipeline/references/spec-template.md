# Spec Template

Use this structure when drafting the design or spec artifact.

## Title

## Problem
- What is changing
- Why this change is needed now

## Scope
- Included behavior
- Included modules or systems

## Non-goals
- Explicitly excluded work

## Current Constraints
- Existing architecture constraints
- Operational or product constraints
- Compatibility requirements

## Proposed Design
- Main flow
- Key boundaries and ownership
- Interfaces, contracts, or data shape changes
- State transition or lifecycle impacts

## Risks
- Main technical risks
- Main rollout or migration risks
- Failure modes worth calling out

## Validation Plan
- Unit or local validation
- Integration validation
- Manual checks if required

## Rollout and Compatibility
- Migration or backfill needs
- Compatibility with existing clients, jobs, or data
- Rollback assumptions

## Open Questions
- Only keep questions that materially affect implementation or correctness
