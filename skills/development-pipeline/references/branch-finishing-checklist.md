# Branch Finishing Checklist

Use this file before declaring the branch finished.

- is the branch state clean enough for review or handoff
- are commits grouped by reviewable unit instead of accidental churn
- does the final summary explain what changed, how it was validated, and what remains risky
- are migration or rollout notes included when needed
- are deferred items clearly marked as deferred instead of silently omitted
- are branch-specific temporary artifacts, debug leftovers, or local-only hacks removed
- is the result ready for merge, review, or explicit downstream handoff
