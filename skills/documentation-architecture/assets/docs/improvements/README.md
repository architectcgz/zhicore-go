# Improvements

This folder tracks improvement items that agents noticed but that are not yet fully handled by code, documentation, project rules, skills, or agent policies.

## Status Folders

- `not-impl/`: accepted or plausible improvement ideas that are not implemented.
- `implemented/`: improvements already implemented in code, docs, policy, skill, or checks.
- `agent-recorded/`: items captured in an agent policy or skill but not otherwise changed.
- `rejected/`: ideas that were considered and explicitly rejected.
- `archived/`: obsolete or superseded items kept only for history.

## Rules

- Do not use this folder as a general task backlog. Ready-to-execute work belongs in `docs/todo/` or `docs/plan/`.
- Every item should name the problem, affected area, proposed direction, current status, and promotion target.
- Once an item is fully handled elsewhere, move or summarize it under `implemented/` or remove the long body if Git history is enough.
