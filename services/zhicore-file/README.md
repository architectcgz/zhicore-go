# zhicore-file

`zhicore-file` 是文件服务的 Go 目标服务模块。

服务职责：

- 提供统一上传入口、文件类型和大小校验、访问级别参数转换、批量上传、URL 获取和删除。
- 管理文件 ID、文件元数据、对象存储路径、访问级别、临时 / 未绑定文件状态和后续清理任务。
- 为用户头像、文章封面、评论图片、评论语音等场景提供统一文件入口。
- 后续对象存储实现优先落在本服务适配器内，MinIO 是 `zhicore-file` 的基础设施依赖，不是另一个业务服务。

数据归属：

- `zhicore-file` 拥有文件元数据、对象存储 key、URL 解析、秒传、分片、删除和清理语义。
- 其他业务服务只保存 `fileId` 引用和自身业务绑定关系，不保存 CDN URL、对象存储 key 或签名 URL 作为长期事实。

Go 设计注意点：

- 其他服务只保存 `fileId` / `file_id` 引用，不拥有文件资源本身。
- 头像、封面、评论媒体等业务归属仍属于包含它们的业务服务。
- 文件 metadata schema、MinIO adapter、引用确认和 GC worker 必须在本服务内收口，不通过业务服务反向持有对象存储细节。
