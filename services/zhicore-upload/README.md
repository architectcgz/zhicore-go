# zhicore-upload

`zhicore-upload` 是上传服务的 Go 目标服务模块。

服务职责：

- 提供统一上传入口、文件类型和大小校验、访问级别参数转换、批量上传、URL 获取和删除代理。
- 为用户头像、文章封面、评论图片、评论语音等场景提供统一文件入口。

数据归属：

- 当前 Go 第一阶段不拥有本地文件元数据表。
- 文件元数据、对象存储路径、秒传、分片和 CDN URL 仍归外部 File Service。

Go 设计注意点：

- 其他服务只保存 `file_id` 引用，不拥有文件资源本身。
- 头像、封面、评论媒体等业务归属仍属于包含它们的业务服务。
- 如果后续确认 Upload 需要拥有自己的文件元数据表，必须先更新数据归属和 migration 设计。
