# 面试录音复盘提示词

本文件用于把面试录音转写后的 Markdown 当作输入，做结构化复盘、能力差距分析、表达优化和二次模拟。

建议使用方式：

1. 先用“完整面试复盘提示词”做全局分析。
2. 再针对技术题、项目题、行为题分别使用专项提示词。
3. 最后用“改进计划提示词”和“二次模拟提示词”形成下一轮训练材料。

通用约束：

- 只基于我提供的面试转写稿分析，不要编造面试中没有出现的信息。
- 引用判断依据时，请摘录原文片段或概括对应上下文。
- 如果转写稿有识别错误、语义不完整或上下文缺失，请明确标记“不确定”。
- 输出要直接、具体、可执行，不要只写泛泛建议。

【我的背景】

{{简要填写学历、年级、工作年限、主要技术栈、目标公司或目标岗位层级}}

【面试转写稿】

```markdown
{{粘贴面试转写稿}}
```

【复盘要求】

1. 只基于转写稿分析，不要脑补。
2. 先判断整体面试表现，再拆解具体问题。
3. 区分“知识不会”“表达没讲清”“答非所问”“缺少业务或工程深度”“临场沟通问题”。
4. 对每个主要问题给出：

6. 如果发现我在某些地方表现不错，也要指出具体好在哪里。

请严格按照以下结构输出：

- 通过概率：高 / 中 / 低

## 2. 面试问题清单

- 问题原文或转写稿中的问题概括

- 面试官考察点
- 我的回答摘要
- 评价：好 / 一般 / 有明显问题

## 3. 逐题复盘

对每个关键问题做深度复盘。

格式：

### 问题 N：{{问题标题}}

原始问题：

我的回答：

面试官真正想听到的内容：

我的回答命中了什么：

我的回答缺失了什么：

问题本质：

改进后的回答结构：

优化版回答：

可能追问：

追问回答建议：

## 4. 能力维度诊断

请按以下维度给出 1-5 分评分，并说明依据：

- 技术基础
- 项目理解
- 工程实践
- 问题拆解
- 表达结构
- 沟通互动
- 追问承接
- 岗位匹配度

## 5. 失分模式归纳

请归纳我在这场面试中反复出现的失分模式。

每个模式包括：

- 现象
- 转写稿证据
- 根因
- 下次避免方式
- 训练方法

## 6. 优化后的自我介绍和项目介绍

如果转写稿中出现自我介绍或项目介绍，请帮我重写为更适合面试的版本。

要求：

- 控制在 1-2 分钟
- 突出岗位匹配点
- 有技术深度入口
- 能自然引导面试官追问我准备好的内容

## 7. 下一轮面试准备清单

请给出优先级排序：

- 必须补的知识点
- 必须重写的项目表达
- 必须练习的回答
- 可以放弃深挖的低优先级内容

每一项都说明为什么。

# 七、二次模拟面试提示词

你是一名严格但公平的面试官。

请基于我的上一次面试转写稿和复盘结果，对我进行二次模拟面试。

【目标岗位】

{{填写岗位}}

【上次面试转写稿】

```markdown
{{粘贴面试转写稿}}
```

【上次复盘结果】

```markdown
{{粘贴复盘结果}}
```

请按照以下方式进行：

1. 先提出第一个问题，不要一次性给出所有问题。
2. 每次我回答后，你再评价我的回答。
3. 评价必须包括：
   - 是否回答了问题
   - 结构是否清楚
   - 技术点是否准确
   - 有哪些失分点
   - 如何改成更好的回答
4. 然后继续追问。
5. 优先追问我上次面试暴露出来的短板。
6. 如果我答得好，可以提高问题难度。
7. 如果我答得差，请先指出最关键的问题，再给我一次重答机会。

现在请开始模拟面试，只问第一个问题。
