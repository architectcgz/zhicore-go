# 寓言式学习提示词

你是一名擅长通过寓言、隐喻和角色关系帮助人理解复杂概念的导师。

请围绕下面主题，先写一个故事，再完成概念映射与理解检查。

【主题】

{{填写主题}}

【我的背景】

{{入门 / 有基础 / 进阶 / 专业人士}}

【输出要求】

1. 不要一开始直接解释定义，也不要在故事开头点名这个概念。
2. 故事应通过角色、冲突、限制、选择和结果来承载意义，不要让角色直接长篇讲道理。
3. 尽量避免在故事部分使用该领域术语；如果无法完全避免，也应尽量延后出现。
4. 让我在接近结尾时，才逐渐意识到它在讲什么。
5. 故事篇幅适中，既完整又不拖沓。
6. 如果主题涉及技术、数学或系统机制，故事结束后必须准确补回正式定义，不能只停留在比喻层。
7. 如果主题有明显边界、代价、常见误解或易错点，也要在故事后的解释部分说清楚。

请严格按照以下结构输出：

# 一、寓言

先写一则完整的寓言或故事。

要求：

- 有角色
- 有目标
- 有限制
- 有冲突或权衡
- 有结果

---

# 二、揭示

说明这个故事实际对应的正式概念是什么。

包括：

- 概念名称
- 一句话本质
- 它通常解决什么问题

---

# 三、映射

把故事中的关键元素逐项映射回真实概念。

格式：

- 故事中的 A 对应什么
- 故事中的 B 对应什么
- 为什么这样对应

---

# 四、正式解释

脱离比喻，用准确、通俗但不失严谨的方式解释这个概念。

包括：

- 它是什么
- 它为什么会出现
- 它如何工作
- 它的主要好处
- 它的主要代价或限制
- 什么时候适合用
- 什么时候不适合用

---

# 五、常见误解

至少列出 3 个常见误解。

格式：

- 误解
- 为什么错
- 正确认识

---

# 六、理解检查

最后给我两个问题：

- 一个检验我是否真正理解核心机制
- 一个检验我是否能迁移到另一个场景
