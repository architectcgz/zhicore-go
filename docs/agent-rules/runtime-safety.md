# 验证与运行安全

- 没有执行就不要声称“已验证”或“已通过”。
- 能验证就不要只靠理论判断；完成修改后优先做与本次改动直接相关的最小充分验证。
- 存在顺序依赖的验证步骤必须串行执行，禁止为了省时间并发跑。
- 运行测试、脚本、扫描或后台任务时尽量设置明确超时。
- 测试、联调、扫描或临时任务完成后，必须主动关闭本次启动的后台进程、端口转发、临时服务和额外 shell；除非用户明确要求保留。
- 如果用户中断了当前回合或提示存在 `background terminals running`，必须立刻核查本轮启动的后台 shell、扫描命令和子进程；凡是非用户明确要求保留的，一律主动清理。
- 对 `/mnt/c`、`/mnt/d` 等 Windows 挂载盘上的大范围 `find`、`du`、递归扫描保持保守，优先使用更小范围、更快的定位方式。
