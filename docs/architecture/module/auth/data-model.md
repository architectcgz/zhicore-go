# Auth 数据模型设计

本文记录 Auth 模块 PostgreSQL 数据模型草案。正式可执行 schema 仍以后续 `services/zhicore-auth/migrations/` 中的 SQL migration 为准；在生成 migration 前，以本文约束表、字段、索引和保留策略。

## 设计原则

- PostgreSQL 是账号、凭证、refresh session、安全 operation、审计和 outbox 的真相源。
- Redis 只保存 Gateway 可见的撤销、版本、principal cache、refresh session cache 和限流投影，见 `redis-keys.md`。
- 表名统一使用 `auth_*` 前缀，避免与其他服务或未来共享库中的通用 `accounts`、`outbox_events` 混淆。
- 不创建跨服务外键。`user_id` 是 User 服务返回的内部 ID 引用，由 Auth 保存映射但不拥有 User profile。
- 所有持久化时间使用 `TIMESTAMPTZ`；响应时间序列化规则仍按 HTTP contract。
- password、password hash、refresh token、access token、cookie、Authorization header 不写入审计 metadata、outbox payload、Redis key 或普通日志。

## 表总览

| 表 | 归属 | 作用 |
| --- | --- | --- |
| `auth_accounts` | Auth | 账号生命周期、登录标识、User 映射、版本号和登录元数据。 |
| `auth_password_credentials` | Auth | 密码 hash、密码更新时间和凭证版本。 |
| `auth_account_roles` | Auth | 账号角色授予、撤销和审计来源。 |
| `auth_refresh_sessions` | Auth | 浏览器/设备级 refresh session、当前 refresh token hash、过期和撤销状态。 |
| `auth_used_refresh_tokens` | Auth | 已 rotation 使用过的 refresh token 记录，用于 replay 检测。 |
| `auth_email_verifications` | Auth | 邮箱验证码发送、校验、尝试次数和发送状态。 |
| `auth_verification_tokens` | Auth | `emailVerificationToken`、`passwordResetToken` 等短期一次性不透明 token 的 hash。 |
| `auth_security_operations` | Auth | Redis 投影写失败或安全同步处理中时的可查询 operation。 |
| `auth_audit_logs` | Auth | Auth 本地安全审计日志。 |
| `auth_outbox_events` | Auth | Auth 集成事件 transactional outbox。 |

是否把 email verification / reset token 表放进首个 migration，由实现切片决定；但表结构和 owner 以本文为准。

## `auth_accounts`

账号生命周期和认证主体映射表。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键，`GENERATED BY DEFAULT AS IDENTITY`。 |
| `user_id` | `BIGINT NULL` | User 服务返回的内部用户 ID；账号 `ACTIVE` 后必须存在。 |
| `email` | `VARCHAR(254)` | 展示用 email，保留用户输入规范化后的展示值。 |
| `email_normalized` | `VARCHAR(254)` | 登录查询和唯一约束使用，`trim + lower-case`。 |
| `status` | `VARCHAR(32)` | `PENDING_PROFILE`、`ACTIVE`、`DISABLED`、`BANNED`、`EXPIRED`。 |
| `session_version` | `BIGINT` | 登录态有效性版本，默认 `1`。 |
| `principal_version` | `BIGINT` | 认证主体快照版本，默认 `1`。 |
| `registration_nickname` | `VARCHAR(64) NULL` | pending 注册阶段临时保存 nickname；激活后可清空。 |
| `pending_expires_at` | `TIMESTAMPTZ NULL` | `PENDING_PROFILE` 过期时间。 |
| `locked_until` | `TIMESTAMPTZ NULL` | 登录失败等风控临时锁定截止时间。 |
| `lock_reason` | `VARCHAR(64) NULL` | 临时锁定原因码。 |
| `last_login_at` | `TIMESTAMPTZ NULL` | 最近一次登录成功时间。 |
| `last_login_ip` | `INET NULL` | 最近一次登录成功的可信 client IP；不对外响应。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |
| `updated_at` | `TIMESTAMPTZ` | 更新时间。 |

约束和索引：

- `id` 主键。
- `user_id` 对非空值唯一。
- `email_normalized` 对占用状态局部唯一：`PENDING_PROFILE`、`ACTIVE`、`DISABLED`、`BANNED` 占用；`EXPIRED` 不占用。
- 查询索引：`email_normalized`、`status`、`pending_expires_at`、`locked_until`。

状态规则：

- `PENDING_PROFILE` 不签发 token，不进入 principal。
- `ACTIVE` 必须已有 `user_id`。
- `DISABLED` 和 `BANNED` 都不可登录、不可 refresh，且继续占用 email。
- `EXPIRED` 只表示 pending 超时废弃，不可登录、不占用 email。
- 临时锁定不改变 `status`，只使用 `locked_until/lock_reason`。

## `auth_password_credentials`

密码凭证表。password hash 不放 `auth_accounts`，减少敏感字段扩散。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `account_id` | `BIGINT` | Auth account ID。 |
| `password_hash` | `VARCHAR(255)` | PHC 格式或 bcrypt 格式；不保存明文。 |
| `credential_version` | `BIGINT` | 凭证版本，默认 `1`。 |
| `password_updated_at` | `TIMESTAMPTZ` | 最近一次密码更新时间。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |
| `updated_at` | `TIMESTAMPTZ` | 更新时间。 |

约束和索引：

- `account_id` 唯一。
- `account_id` 可加服务内外键到 `auth_accounts(id)`；是否启用由 migration 切片决定。禁止跨服务外键。

## `auth_account_roles`

角色授予历史表。首期不把 roles 存成 `auth_accounts.roles jsonb`。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `account_id` | `BIGINT` | 目标账号。 |
| `role` | `VARCHAR(64)` | 例如 `ROLE_USER`、`ROLE_ADMIN`、`ROLE_MODERATOR`。 |
| `granted_at` | `TIMESTAMPTZ` | 授予时间。 |
| `granted_by_account_id` | `BIGINT NULL` | 操作者账号；系统/bootstrap 可为空。 |
| `grant_reason` | `VARCHAR(128) NULL` | 结构化原因码。 |
| `revoked_at` | `TIMESTAMPTZ NULL` | 撤销时间；为空表示当前有效。 |
| `revoked_by_account_id` | `BIGINT NULL` | 撤销操作者。 |
| `revoke_reason` | `VARCHAR(128) NULL` | 结构化撤销原因码。 |

约束和索引：

- 同一个账号同一个 active role 只能有一条：`UNIQUE(account_id, role) WHERE revoked_at IS NULL`。
- 查询当前 principal 使用 `(account_id, revoked_at)`。
- role 变化必须增加目标账号 `principal_version`，写 audit，并写 `auth.account.role_changed.v1` outbox。

首期不强制单独 `auth_roles` 参考表。需要 seed 角色时，可在 migration 中使用 `CHECK` 或后续引入参考表；不影响 `auth_account_roles` 作为角色事实表。

## `auth_refresh_sessions`

refresh session 真相源。一个 session 表示一次登录设备/浏览器会话。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `session_id` | `VARCHAR(64)` | 对外/Token claim 使用的稳定 session ID，高熵随机或 ID 生成器生成。 |
| `account_id` | `BIGINT` | 所属账号。 |
| `current_token_id` | `VARCHAR(64)` | 当前 refresh token 的非秘密定位符。 |
| `current_token_hash` | `VARCHAR(128)` | `HMAC-SHA-256(serverPepper, normalizedRefreshToken)` 等结果。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |
| `last_used_at` | `TIMESTAMPTZ NULL` | 最近 refresh 成功时间。 |
| `expires_at` | `TIMESTAMPTZ` | refresh session 过期时间。 |
| `revoked_at` | `TIMESTAMPTZ NULL` | 撤销时间。 |
| `revoked_reason` | `VARCHAR(64) NULL` | `LOGOUT_CURRENT`、`REVOKE_SESSION`、`PASSWORD_CHANGED` 等。 |
| `revoked_by_account_id` | `BIGINT NULL` | 操作者账号。 |
| `last_access_jti` | `VARCHAR(128) NULL` | 最近已知 access token `jti`，用于当前 session 撤销时写黑名单。 |
| `user_agent` | `TEXT NULL` | 原始 UA，仅用于设备展示和安全排查。 |
| `device_label` | `VARCHAR(128) NULL` | 解析后的设备展示名称。 |
| `created_ip` | `INET NULL` | 创建 session 的可信 client IP。 |
| `last_seen_ip` | `INET NULL` | 最近使用的可信 client IP。 |

约束和索引：

- `session_id` 唯一。
- `(account_id, revoked_at, expires_at)` 支持 session list。
- `(account_id, expires_at)` 支持账号级清理和 revoke all。
- `current_token_id` 可按实现需要加唯一索引；`current_token_hash` 不对外暴露。

rotation 规则：

- `POST /auth/refresh` 按 `session_id` 开事务并锁行。
- 校验 `current_token_id/current_token_hash` 成功后，生成新 refresh token，写 `auth_used_refresh_tokens`，再更新当前 session 的 `current_token_id/current_token_hash/last_used_at`。
- 旧 token 再出现时，若命中 used token 记录，按 replay 处理并吊销当前 session。

## `auth_used_refresh_tokens`

已使用 refresh token 记录，用于区分 replay 与普通 invalid。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `session_id` | `VARCHAR(64)` | 所属 refresh session。 |
| `account_id` | `BIGINT` | 冗余账号 ID，方便审计和清理。 |
| `token_id` | `VARCHAR(64)` | 被 rotation 消费的 token ID。 |
| `token_hash` | `VARCHAR(128)` | 被消费 token 的 hash。 |
| `used_at` | `TIMESTAMPTZ` | 被成功消费时间。 |
| `expires_at` | `TIMESTAMPTZ` | 记录保留到期时间，默认 30 天。 |

约束和索引：

- `token_id` 或 `token_hash` 唯一，具体以 token 格式和 hash 策略决定。
- `(session_id, token_id)` 支持 replay 查询。
- `expires_at` 支持清理任务。

## `auth_email_verifications`

邮箱验证码生命周期表。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `email_normalized` | `VARCHAR(254)` | 绑定邮箱。 |
| `purpose` | `VARCHAR(32)` | `register`、`reset_password`、后续 `change_email`。 |
| `code_hash` | `VARCHAR(128)` | 验证码 hash，不保存明文 code。 |
| `status` | `VARCHAR(32)` | `PENDING_SEND`、`SENT`、`SEND_FAILED`、`CONSUMED`、`SUPERSEDED`、`EXPIRED`。 |
| `attempt_count` | `INT` | 校验失败次数。 |
| `max_attempts` | `INT` | 默认 5，配置化。 |
| `expires_at` | `TIMESTAMPTZ` | 默认 10 分钟，配置化。 |
| `sent_at` | `TIMESTAMPTZ NULL` | 发送成功时间。 |
| `consumed_at` | `TIMESTAMPTZ NULL` | 校验成功消费时间。 |
| `request_ip` | `INET NULL` | 可信 client IP。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |

索引：

- `(email_normalized, purpose, created_at DESC)` 查询最新验证码。
- `(email_normalized, purpose, status)` 支持旧验证码 supersede。
- `expires_at` 支持清理。

规则：

- 只有最新 `SENT` 且未过期、未消费、未超过尝试次数的验证码可验证。
- 重新发送成功后，旧的 `SENT` 记录标记 `SUPERSEDED` 或在查询逻辑上失效。

## `auth_verification_tokens`

短期一次性不透明 token 的 hash 存储，例如注册用 `emailVerificationToken` 和找回密码用 `passwordResetToken`。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `token_id` | `VARCHAR(64)` | 非秘密定位符。 |
| `token_hash` | `VARCHAR(128)` | token hash，不保存明文 token。 |
| `email_normalized` | `VARCHAR(254)` | 绑定邮箱。 |
| `purpose` | `VARCHAR(32)` | `register`、`reset_password`、后续 `change_email`。 |
| `issued_from_verification_id` | `BIGINT NULL` | 来源验证码记录。 |
| `expires_at` | `TIMESTAMPTZ` | 默认 10 分钟，配置化。 |
| `consumed_at` | `TIMESTAMPTZ NULL` | 成功使用时间。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |

约束和索引：

- `token_id` 唯一。
- `token_hash` 唯一。
- `(email_normalized, purpose, consumed_at, expires_at)` 支持 register/reset confirm 校验。

## `auth_security_operations`

安全撤销类操作的持久状态。用于 `202 PROCESSING` 后前端查询，也用于 worker 补偿 Redis 投影。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `operation_id` | `VARCHAR(64)` | 对外返回 ID，唯一。 |
| `type` | `VARCHAR(64)` | `LOGOUT_CURRENT`、`REVOKE_SESSION`、`PASSWORD_CHANGE`、`ACCOUNT_BAN` 等。 |
| `account_id` | `BIGINT` | 被影响账号。 |
| `target_session_id` | `VARCHAR(64) NULL` | 目标 session。 |
| `actor_type` | `VARCHAR(32)` | `user`、`admin`、`system`、`service`。 |
| `actor_account_id` | `BIGINT NULL` | 操作者账号。 |
| `status` | `VARCHAR(32)` | `PENDING`、`SUCCEEDED`、`FAILED`。 |
| `retry_after_seconds` | `INT NULL` | 建议前端轮询间隔。 |
| `attempt_count` | `INT` | 补偿尝试次数。 |
| `next_retry_at` | `TIMESTAMPTZ NULL` | worker 下次重试时间。 |
| `last_error_code` | `VARCHAR(64) NULL` | 可展示/可审计错误码。 |
| `last_error_message` | `TEXT NULL` | 内部错误摘要，不放敏感材料。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |
| `updated_at` | `TIMESTAMPTZ` | 更新时间。 |
| `completed_at` | `TIMESTAMPTZ NULL` | 完成时间。 |

索引：

- `operation_id` 唯一。
- `(account_id, created_at DESC)` 支持用户查询自己的 operation。
- `(status, next_retry_at, id)` 支持补偿 worker claim。

## `auth_audit_logs`

Auth 本地安全审计。审计日志不是跨服务事件，也不是普通业务日志。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `event_type` | `VARCHAR(64)` | `LOGIN_SUCCESS`、`LOGIN_FAILED`、`REFRESH_REPLAYED`、`LOGOUT`、`PASSWORD_CHANGED` 等。 |
| `account_id` | `BIGINT NULL` | 被影响账号。 |
| `user_id` | `BIGINT NULL` | 冗余 User ID，方便审计查询。 |
| `operator_account_id` | `BIGINT NULL` | 操作者账号。 |
| `session_id` | `VARCHAR(64) NULL` | 相关 session。 |
| `request_id` | `VARCHAR(64) NULL` | 请求 ID。 |
| `trace_id` | `VARCHAR(64) NULL` | trace ID。 |
| `ip` | `INET NULL` | 可信 client IP；仅限 Auth 安全数据面。 |
| `ip_hash` | `VARCHAR(128) NULL` | 需要脱敏查询时使用。 |
| `user_agent` | `TEXT NULL` | 原始 UA；普通应用日志不得输出完整 UA。 |
| `user_agent_hash` | `VARCHAR(128) NULL` | UA hash。 |
| `metadata_json` | `JSONB NULL` | 结构化白名单 metadata。 |
| `created_at` | `TIMESTAMPTZ` | 创建时间。 |

禁止写入：

- password、password hash。
- access token、refresh token、reset token、验证码。
- cookie、Authorization header、完整请求体。

索引：

- `(account_id, created_at DESC)`。
- `(event_type, created_at DESC)`。
- `(request_id)`。
- `(ip_hash, created_at DESC)`。

默认保留 1 年，保留期配置化；删除/归档只能由后台 retention job 执行。

## `auth_outbox_events`

Auth producer outbox。业务状态变更和 outbox 写入必须在同一 PostgreSQL 事务内完成。

| 字段 | 类型 | 约束 / 说明 |
| --- | --- | --- |
| `id` | `BIGINT` | 主键。 |
| `event_id` | `VARCHAR(64)` | 全局唯一事件 ID。 |
| `event_type` | `VARCHAR(128)` | 例如 `auth.account.registered.v1`。 |
| `event_version` | `INT` | payload 版本。 |
| `aggregate_type` | `VARCHAR(64)` | `account` 等。 |
| `aggregate_id` | `VARCHAR(64)` | 通常为 `accountId`。 |
| `payload_json` | `JSONB` | 最小必要 payload。 |
| `status` | `VARCHAR(32)` | `PENDING`、`PUBLISHED`、`FAILED`。 |
| `attempt_count` | `INT` | 投递尝试次数。 |
| `next_retry_at` | `TIMESTAMPTZ NULL` | 下次重试时间。 |
| `last_error` | `TEXT NULL` | 内部错误摘要。 |
| `occurred_at` | `TIMESTAMPTZ` | 事件发生时间。 |
| `created_at` | `TIMESTAMPTZ` | outbox 创建时间。 |
| `published_at` | `TIMESTAMPTZ NULL` | 成功投递时间。 |

索引：

- `event_id` 唯一。
- `(status, next_retry_at, id)` 支持 publisher claim。
- `(aggregate_type, aggregate_id, occurred_at)` 支持排查。

## 清理和保留

| 数据 | 默认保留策略 |
| --- | --- |
| `PENDING_PROFILE` account | 超过 `AUTH_PENDING_PROFILE_TTL` 后转 `EXPIRED`，释放 email，但保留历史行。 |
| `auth_used_refresh_tokens` | 默认保留 30 天，过期后硬删除。 |
| revoked / expired refresh sessions | 默认保留 90 天，配置化。 |
| security operations | 默认保留 7-30 天，配置化。 |
| outbox `PUBLISHED` | 默认保留 7-30 天，配置化；`FAILED` 不自动删除。 |
| audit logs | 默认保留 1 年，配置化，可归档或删除。 |

`DISABLED`、`BANNED` 账号本体不由普通清理任务删除，也不自动释放 email。

## Migration 切片建议

正式 SQL migration 建议拆成可审查的几段，而不是一次塞入所有表：

1. `create_auth_accounts_and_credentials`
2. `create_auth_roles_and_sessions`
3. `create_auth_verification_tables`
4. `create_auth_security_operations_and_audit`
5. `create_auth_outbox`

每个 migration pair 必须放在 `services/zhicore-auth/migrations/`，遵循 `docs/architecture/migrations.md`；不要新增服务级 `schema/` 目录作为第二事实源。
