# Content 页面设计

本文记录 Content 页面相关的前端草稿、页面分区、接口编排、加载状态和降级规则。它是产品 / 前端交互设计事实源，不替代 HTTP 字段级 contract；字段、错误码和 path 仍以 `services/zhicore-content/api/http/` 为准。

当前状态：本文固定页面草案和加载逻辑，不表示前端已经实现。

## 设计原则

- 文章是主资源，engagement 是附加资源。页面必须先确认文章或列表可用，再加载点赞、收藏和当前用户状态。
- 正文、标题、作者和发布时间优先于互动信息。互动失败不能让已可读文章变成不可读。
- 不能把未知互动状态展示成未点赞 / 未收藏；`null + degraded=true` 显示为中性态或轻量禁用态。
- 浏览页面不使用弹窗作为默认错误表达。主资源错误显示页面级空态；附加资源错误只影响局部区域。
- 作者工作台和管理运维页面是任务型界面，优先清晰、密度、可恢复操作和明确状态，不做营销式 hero 或装饰性卡片。
- 编辑器页面必须优先保护用户输入。保存失败、发布失败、冲突和网络中断都不能让本地草稿无提示丢失。
- 加载态优先使用骨架和占位，避免页面跳动；计数和按钮区域必须保留稳定尺寸。
- 所有前端文案应走 i18n 文案 key；本文只记录中文语义，不把中文硬编码成协议事实。

## 页面范围

本文覆盖：

- 公开文章列表页。
- 作者主页文章列表。
- 标签文章列表。
- 文章详情页。
- 详情页互动栏。
- 列表页互动摘要。
- 作者工作台文章列表和草稿列表。
- 草稿编辑器、正文 blocks 编辑、保存状态。
- 发布、撤回、定时发布、删除、恢复等生命周期页面动作。
- 标签浏览、标签搜索和热门标签页。
- 管理端内容列表、管理删除和 outbox 运维页面。

本文不覆盖：

- Search / Ranking 独立页面。热榜页面如果使用 Content 摘要，也应遵守本文的 engagement 二阶段加载原则。
- 评论树完整体验。评论列表、评论点赞和回复编辑归 Comment 页面设计。

## 公共页面模型

### 资源分层

| 层级 | 内容 | 失败处理 |
| --- | --- | --- |
| 主资源 | 文章列表、文章详情、正文、作者快照、发布时间、标签 | 页面级错误或空态；不继续加载依附于该资源的 engagement。 |
| 附加资源 | engagement、viewer 状态、reader presence、头像展示 URL 等派生展示字段 | 局部降级；主资源继续展示。 |
| 用户动作 | 点赞、取消点赞、收藏、取消收藏 | 成功后更新本地互动区；失败时回滚乐观态或保持原状态，并显示局部提示。 |
| 作者动作 | 保存草稿、发布、撤回、定时、删除、恢复、标签维护 | 操作级 pending 和错误；保留可恢复输入，不用页面级错误覆盖编辑状态。 |
| 管理动作 | 管理查询、管理删除、outbox retry | 表格局部 pending、审计原因输入、危险操作确认和可追踪错误。 |

### Engagement 状态到 UI 的映射

| API 状态 | UI 状态 | 行为 |
| --- | --- | --- |
| `liked=true` | 点赞按钮选中 | 允许点击取消点赞。 |
| `liked=false` | 点赞按钮未选中 | 允许点击点赞。 |
| `liked=null` + `degraded=true` | 点赞按钮中性 / 轻量禁用 | 不显示为未点赞；可提示“互动状态暂不可用”。 |
| `favorited=true` | 收藏按钮选中 | 允许点击取消收藏。 |
| `favorited=false` | 收藏按钮未选中 | 允许点击收藏。 |
| `favorited=null` + `degraded=true` | 收藏按钮中性 / 轻量禁用 | 不显示为未收藏；可提示“收藏状态暂不可用”。 |

计数加载失败时保留占位，例如 `--`；不要显示 `0`，因为 `0` 是确定计数。

### 登录态

- 匿名用户可以阅读公开文章和查看统计。
- 匿名用户看到未选中的点赞 / 收藏按钮时，点击后进入登录引导，而不是调用命令接口。
- 登录用户加载详情后请求 viewer 状态。
- 登录态变化后，当前页面应重新请求 engagement；不能沿用匿名状态或旧用户状态。

## 公开文章列表页

### 页面草稿

```text
┌────────────────────────────────────────────┐
│ Top nav / search / user entry              │
├────────────────────────────────────────────┤
│ Filter row: category · tags · sort latest  │
├────────────────────────────────────────────┤
│ Article list                               │
│                                            │
│  Title                                     │
│  summary excerpt                           │
│  author · publishedAt · tags               │
│  view · like · favorite · comment          │
│                                            │
│  ...                                       │
├────────────────────────────────────────────┤
│ Load more / infinite sentinel              │
└────────────────────────────────────────────┘
```

列表项不要依赖 viewer 状态才能渲染。第一屏先展示标题、摘要、作者、发布时间和统计；登录用户的 liked / favorited 高亮可在列表数据返回后或批量状态接口返回后再补。

### 加载逻辑

1. 页面进入时调用 `GET /api/v1/posts`，携带 `limit`、`cursor`、`authorId`、`tag`、`categoryId` 等过滤参数。
2. 主资源加载中显示列表骨架，骨架行数与当前 viewport 预计可见行数一致。
3. 主资源成功后渲染文章列表。
4. 如果用户已登录，收集当前页 `postId`，调用 `POST /api/v1/posts/engagement/batch-status`。
5. 批量状态返回后只更新当前页仍存在的列表项；用户翻页或过滤变化后，过期响应必须丢弃。
6. `nextCursor` 存在时显示加载更多或触发下一页 sentinel。

### 状态处理

| 场景 | 页面行为 |
| --- | --- |
| 列表主资源加载失败 | 显示页面级错误和重试按钮，不请求 batch engagement。 |
| 列表为空 | 显示空态；空态文案说明当前过滤下暂无文章，不展示 engagement。 |
| batch engagement 失败 | 列表继续展示；互动按钮保持中性态或只显示统计，不逐条弹错。 |
| 某些 item `degraded=true` | 仅对应 item 的 liked / favorited 显示 unknown，不影响其他 item。 |
| 加载更多失败 | 保留已加载列表，在底部显示局部重试。 |

## 作者主页文章列表

作者主页文章列表复用公开文章列表的主资源和互动加载模型。

差异：

- 使用 `GET /api/v1/posts?authorId={authorId}`。
- 作者资料事实来自 User，不由 Content 页面设计定义。
- 作者资料加载失败时可以继续展示文章列表中已有的作者快照，但作者主页头部应局部降级。
- authorId 变化时必须取消或丢弃旧请求，避免旧作者文章或 engagement 写入新页面。

## 标签文章列表

标签文章列表复用公开文章列表模型。

差异：

- 先加载标签详情或使用路由中的 `slug` 展示轻量标题。
- 调用 `GET /api/v1/tags/{slug}/posts`。
- 标签不存在时显示页面级 404，不请求文章列表和 engagement。
- 标签文章列表的排序第一阶段仍以最新发布为主；热门排序归 Ranking，不在 Content 页面中伪造。

## 文章详情页

### 页面草稿

```text
┌────────────────────────────────────────────┐
│ Top nav / back / user entry                │
├────────────────────────────────────────────┤
│ Article header                             │
│  title                                     │
│  author · publishedAt · tags               │
│  optional cover                            │
├────────────────────────────────────────────┤
│ Body blocks                                │
│  paragraphs, headings, media, code, table  │
├────────────────────────────────────────────┤
│ Engagement bar                             │
│  like · favorite · comments · share        │
├────────────────────────────────────────────┤
│ Related / comments entry                   │
└────────────────────────────────────────────┘
```

桌面端可把 engagement bar 做成正文下方固定宽度工具栏或右侧窄栏；移动端应放在正文下方或底部操作条。按钮尺寸、计数宽度和 loading 占位必须稳定，不能因计数变化造成布局跳动。

### 加载逻辑

1. 页面进入时只请求主资源：`GET /api/v1/posts/{postId}`。
2. 主资源加载中显示文章标题、作者、正文段落骨架；不显示假的点赞 / 收藏状态。
3. 主资源失败时显示页面级错误：
   - `4001`：文章不存在、已删除或不可见。
   - `4018` / `4024`：正文不可用或不一致，提示稍后重试。
   - `1004`：服务暂时不可用。
4. 主资源成功后渲染标题、作者、标签、正文和基础统计。
5. 主资源成功后再请求 `GET /api/v1/posts/{postId}/engagement`。
6. Engagement 加载中时，互动栏展示骨架或中性按钮，不阻塞阅读。
7. Engagement 成功后更新计数和 viewer 状态。
8. Engagement 失败时，只降级互动栏；文章正文保持可读。

### 文章不可用时

文章详情不可用时，engagement 没有独立展示意义。前端必须：

- 不请求 `GET /api/v1/posts/{postId}/engagement`。
- 不展示点赞、收藏、评论计数等互动栏。
- 不允许触发点赞 / 收藏命令。
- 页面焦点落在主错误区域，提供返回列表或重试入口。

后端仍会在 engagement endpoint 校验文章可见性；前端调用顺序只是减少无意义请求和错误噪声。

## 详情页互动栏

### 草稿

```text
┌────────────────────────────────────────────┐
│ [Like icon] 10    [Bookmark icon] 5        │
│ [Comment icon] 3  [Share icon]             │
└────────────────────────────────────────────┘
```

按钮规则：

- 使用图标按钮加数字，图标需要可访问标签。
- `liked=true` / `favorited=true` 时使用 selected 状态。
- `liked=false` / `favorited=false` 时使用 default 状态。
- `null + degraded=true` 时使用 unknown 状态，不用 selected，也不用 default 的“可确定未选中”语义。
- 命令执行中按钮进入 loading / disabled，保留原计数，避免重复点击。

### 点赞命令

点击点赞：

1. 未登录：打开登录入口或跳转登录，不调用 `PUT /like`。
2. 当前状态 unknown：不做乐观更新，可提示“互动状态暂不可用”或先尝试重新加载 engagement。
3. 当前状态 `false`：调用 `PUT /api/v1/posts/{postId}/like`。
4. 成功：使用响应里的 `liked` 和 `likeCount` 更新互动栏。
5. 失败：保持原状态，按错误码显示局部提示。

取消点赞同理调用 `DELETE /api/v1/posts/{postId}/like`。收藏 / 取消收藏同构。

### 错误文案语义

| 错误 | 前端语义 |
| --- | --- |
| `2006` | 需要登录后操作。 |
| `4001` | 文章已不可用，刷新或返回列表。 |
| `1003` | 操作太频繁，请稍后再试。 |
| `1004` | 互动服务暂不可用，请稍后再试。 |
| network timeout | 网络异常，请稍后重试。 |

命令失败不要把按钮切到相反状态；只有服务确认成功后才更新事实状态。

## 列表页互动摘要

列表页互动摘要只承载轻量信息：

- 统计数字可直接来自 `PostSummary.stats`。
- liked / favorited 高亮来自 `batch-status`。
- batch 失败时保留统计，隐藏或中性化 viewer 高亮。
- 不在列表页弹出逐项错误。

列表项点击进入详情时，详情页必须重新按详情页逻辑加载 engagement，不能把列表页的 batch 状态当成详情页最终事实。

## Reader Presence 展示

Reader presence 是附加能力，不影响文章阅读：

- 详情页主体加载成功后再注册或刷新 reader session。
- Presence 加载中不占用主要视觉层级。
- Redis 不可用时，presence 显示为空摘要或隐藏；不显示页面级错误。
- Presence 返回 `degraded=true` 时，只影响在线读者摘要，不影响 engagement 或正文。

## 作者工作台文章列表

### 页面草稿

```text
┌────────────────────────────────────────────┐
│ Workspace nav: All · Draft · Published     │
│                Scheduled · Deleted         │
├────────────────────────────────────────────┤
│ Toolbar: search/filter · New post          │
├────────────────────────────────────────────┤
│ Row: title · status · updatedAt · actions  │
│ Row: title · status · updatedAt · actions  │
└────────────────────────────────────────────┘
```

作者工作台是任务型页面，列表密度应高于公开浏览页。每一行优先展示标题、状态、最近更新时间、发布时间 / 定时发布时间和主要操作；摘要、封面和统计只能作为辅助信息。

### 加载逻辑

1. 进入页面先确认登录态；缺少登录态时进入登录流程，不请求 `GET /api/v1/me/posts`。
2. 调用 `GET /api/v1/me/posts?status={status}&cursor={cursor}&limit={limit}`。
3. 加载中显示表格或列表骨架，保留筛选栏。
4. 成功后按 `updated_at DESC, public_id DESC` 展示。
5. 切换 status tab 时取消或丢弃旧请求。
6. `nextCursor` 存在时显示加载更多。

### 状态和操作

| 状态 | 行内主操作 | 次级操作 |
| --- | --- | --- |
| `DRAFT` | 继续编辑 | 删除草稿、发布、定时发布 |
| `PUBLISHED` | 查看文章 | 编辑草稿、撤回、删除 |
| `SCHEDULED` | 查看定时 | 修改草稿、取消定时、删除 |
| `DELETED` | 查看摘要 | 恢复 |

空态按当前筛选区分：草稿为空时引导创建文章；已发布为空时提示尚未发布；已删除为空时只显示安静空态，不放大操作。

## 草稿编辑器

### 页面草稿

```text
┌────────────────────────────────────────────┐
│ Editor top bar: back · save state · preview│
│                 publish / schedule menu    │
├────────────────────────────────────────────┤
│ Title input                                │
│ Summary / cover / category / tags panel    │
├────────────────────────────────────────────┤
│ Body block editor                          │
│ paragraph · heading · media · table · code │
└────────────────────────────────────────────┘
```

编辑器首要目标是保护输入。页面不应因为保存失败清空编辑内容，也不应在发布失败时回退本地未保存变更。

### 初始加载

1. 新建文章：调用 `POST /api/v1/posts` 创建空草稿，成功后进入编辑器路由。
2. 编辑已有文章：调用 `GET /api/v1/posts/{postId}/draft`。
3. 加载中显示编辑器骨架；标题、元数据面板、正文区域都保留稳定位置。
4. `2006` 进入登录；`2008` 显示无权访问；`4001` / `4004` 显示文章不可编辑。

### 保存逻辑

元数据和正文分开保存：

- 标题、摘要、封面、分类、标签走 `PATCH /draft/meta`。
- 正文 blocks 走 `PUT /draft/body`。
- 保存请求必须携带当前 `basePostVersion`；正文保存还携带 `baseDraftBodyId` 和 `baseDraftBodyHash`。
- 保存成功后更新本地 `postVersion`、`draftBodyId`、`draftBodyHash` 和 `savedAt`。
- 保存失败时保留本地编辑内容，显示局部错误和重试入口。

Autosave 可以做防抖和队列，但同一资源同一时刻只能有一个正文保存请求在途。新的编辑发生在保存请求期间时，保存成功后仍需标记“有未保存更改”，不能误显示已保存。

### 冲突和错误

| 错误 | 页面行为 |
| --- | --- |
| `4017` 乐观锁冲突 | 显示冲突提示，提供重新加载远端、保留本地副本和手动合并入口。 |
| `4013` / `4014` blocks 非法 | 在对应 block 附近标记错误，保留用户输入。 |
| `4021` 媒体引用不可用 | 标记对应媒体 block 或封面字段，不清空正文。 |
| `1004` 服务暂不可用 | 保留未保存状态，允许稍后重试。 |
| network timeout | 保留本地草稿状态，显示离线 / 网络异常提示。 |

## 发布和生命周期操作

### 发布确认

发布不是普通保存按钮的副作用。点击发布时应进入确认面板或确认页，展示：

- 标题、摘要、封面、标签和正文摘要。
- 当前 `postVersion`、`draftBodyId`、`draftBodyHash` 对应的保存状态。
- 发布前校验结果，例如标题为空、正文过短、媒体引用不可用。

只有当前草稿已保存且没有阻断错误时，才允许调用 `POST /publish`。

### 发布流程

1. 若有未保存变更，先提示保存草稿；不能直接发布旧 `draftBodyId`。
2. 调用 `POST /api/v1/posts/{postId}/publish`。
3. 发布中禁用发布按钮，但允许用户继续阅读确认信息。
4. 成功后跳转公开详情页或显示发布成功状态，使用返回的 `publishedAt`。
5. 失败时停留在编辑器或确认面板，按错误定位问题。

### 定时发布、撤回、删除和恢复

| 操作 | 页面入口 | 关键 UI 状态 |
| --- | --- | --- |
| 定时发布 | 发布菜单 | 日期时间选择、时区展示、草稿保存状态校验。 |
| 取消定时 | 工作台行操作或详情状态条 | 操作级 pending，成功后状态回到 `DRAFT`。 |
| 撤回发布 | 已发布文章的作者操作 | 危险操作确认，成功后公开详情不可见，作者回到工作台。 |
| 作者删除 | 工作台行操作或编辑器菜单 | 危险操作确认，成功后进入 `DELETED` 或返回列表。 |
| 恢复 | 已删除列表行操作 | 成功后状态回到 `DRAFT`，提示继续编辑或发布。 |

危险操作必须说明影响，不使用含糊按钮文案。删除和撤回不能和保存、发布请求并发执行。

## 标签页面和标签输入

### 标签浏览

标签页包含：

- 标签列表：`GET /api/v1/tags`。
- 标签搜索 / 自动补全：`GET /api/v1/tags/search?keyword=...`。
- 热门标签：`GET /api/v1/tags/hot`。
- 标签文章：`GET /api/v1/tags/{slug}/posts`。

标签详情不可用时，不请求标签文章列表。标签文章列表仍按公开列表的 engagement 二阶段加载。

### 编辑器标签输入

编辑器内标签输入应支持：

- 最多 10 个标签的数量提示。
- 输入搜索和候选选择。
- 重复标签去重。
- 保存前本地校验，保存后以后端返回的 `Tag[]` 为准。
- `4012` 分类、话题或标签引用不存在时，在对应字段显示错误。

标签搜索失败不影响正文编辑；只降级候选列表。

## 管理端内容页面

### 内容列表草稿

```text
┌────────────────────────────────────────────┐
│ Admin filters: keyword · status · author   │
├────────────────────────────────────────────┤
│ Table: title · author · status · stats     │
│        createdAt · publishedAt · actions   │
└────────────────────────────────────────────┘
```

管理端使用 page 分页，不使用公开浏览的 cursor 交互。筛选条件变化时重置到第一页。

加载逻辑：

1. 管理员身份缺失时显示无权限，不请求管理接口。
2. 调用 `GET /api/v1/admin/content/posts`。
3. 表格加载中保留筛选栏和表头。
4. 查询失败显示表格级错误，不清空已有筛选。
5. 翻页失败保留当前页数据并显示局部重试。

### 管理删除

管理删除是高风险操作：

- 必须使用确认界面收集可选 `reason`。
- 确认文案说明文章将从公开列表和详情不可见。
- 提交 `DELETE /api/v1/admin/content/posts/{postId}`。
- 成功后更新当前行状态或从当前筛选结果移除。
- `4004` 已删除时显示为已处理，不反复提示失败。

## Outbox 运维页面

### 页面草稿

```text
┌────────────────────────────────────────────┐
│ Filters: status dead/failed · eventType    │
├────────────────────────────────────────────┤
│ Table: eventId · eventType · aggregate     │
│        retryCount · lastError · updatedAt  │
│        retry action                        │
└────────────────────────────────────────────┘
```

Outbox 页面面向运维和管理员，允许信息密度更高，但必须避免泄露敏感 payload。`lastError` 只展示后端允许公开的错误摘要；完整 payload 不在本文定义。

### 加载和 retry

1. 调用 `GET /api/v1/admin/content/outbox-events`。
2. 筛选变化重置 page。
3. 行内 retry 调用 `POST /api/v1/admin/content/outbox-events/{eventId}/retry`。
4. retry 成功后刷新当前行或当前页。
5. retry 命中 `1003` 或 `1004` 时，只显示行内错误，不阻断整个表格。
6. 同一 event retry pending 时禁用该行 retry 按钮。

## 请求编排和竞态

页面实现必须处理这些竞态：

- 路由 `postId` 变化后，旧详情和旧 engagement 响应必须丢弃。
- 登录用户切换后，必须重新加载 viewer engagement。
- 列表过滤条件变化后，旧列表和旧 batch-status 响应必须丢弃。
- 点赞命令执行期间，避免同一按钮重复提交；允许点赞和收藏分别独立 pending。
- 命令成功后，如果随后有旧 engagement 查询返回，不能覆盖更新后的命令结果；需要按请求序号、时间戳或本地 mutation version 防止回退。
- 编辑器保存、发布、定时和删除不能并发修改同一 `postVersion`；高风险操作开始前必须确认没有正文保存请求在途。
- 管理列表筛选、分页和 retry 响应必须按请求序号处理，避免旧筛选结果覆盖新筛选。

## 可访问性和响应式

- 图标按钮必须有 `aria-label`，例如“点赞文章”“取消点赞文章”“收藏文章”“取消收藏文章”。
- `null + degraded=true` 状态按钮必须有可理解标签，例如“点赞状态暂不可用”。
- 键盘用户可以 Tab 到互动按钮；loading / disabled 状态必须可感知。
- 计数变化不应只靠颜色表达。
- 移动端互动栏不遮挡正文和系统安全区域。
- 长标题、长作者名、长标签必须换行或截断，不能覆盖按钮。
- 编辑器工具栏在移动端可以折叠为分组菜单，但保存状态、发布入口和返回入口必须始终可达。
- 管理表格在窄屏下可以折叠为行详情，但筛选、状态和危险操作确认不能丢失。

## 测试和验收

页面实现时至少覆盖：

- 详情页先请求文章，文章失败时不请求 engagement。
- 文章成功、engagement 加载中时正文可读，互动栏稳定占位。
- Engagement 返回 `null + degraded=true` 时，按钮不显示为未点赞 / 未收藏。
- 匿名用户点击点赞进入登录，不调用命令接口。
- 登录用户点赞成功后使用响应计数更新 UI。
- 点赞命令失败后不错误切换按钮状态。
- 列表页 batch-status 失败时列表仍可读。
- 路由切换、登录切换和过滤切换不会写入过期响应。
- 作者工作台按 status 筛选，切换筛选不会展示旧请求结果。
- 编辑器保存失败保留本地输入，乐观锁冲突有明确处理入口。
- 未保存变更存在时不能直接发布旧草稿。
- 发布、撤回、删除、恢复成功后状态和导航符合返回结果。
- 标签搜索失败只降级候选列表，不影响正文编辑。
- 管理删除需要确认，成功后更新表格状态。
- Outbox retry 行内 pending 和错误不阻断整个运维表格。
