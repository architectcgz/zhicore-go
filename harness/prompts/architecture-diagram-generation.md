# 架构图生成输入包模板

## 用途

当需要把一个项目的事实源整理给模型生成架构图、Mermaid、PlantUML、C4 或 draw.io 输入时，使用本模板组织输入。目标是让图表只来自当前事实源，避免把 Draft、历史 review 或旧路径画成当前架构。

项目内如果保留了本地入口文件，先读本文件，再应用项目入口里的局部补充。

## 提示词正文

```text
你要根据我提供的项目事实源生成架构图。请严格遵守：

1. 只使用“允许事实源”中的内容。
2. “禁止来源”只能作为历史背景，不得画成当前架构。
3. 输出 Mermaid 源码，除非我明确要求 PlantUML / C4 / draw.io。
4. 每个节点和每条边都必须能对应到文档、代码、配置或测试依据。
5. 不确定的内容标为“待确认”，不要补不存在的队列、网关、服务、中间件或缓存。
6. 不输出真实 token、cookie、密码、生产连接串、内网敏感地址；必要时使用 `<token>`、`<database-url>`、`<registry-host>` 占位。

目标图类型：
- 例如：容器图 / 组件图 / 时序图 / 状态图 / 数据流图 / 部署图

允许事实源：
- <approved-doc-paths>
- <approved-code-paths>
- <approved-config-paths>
- <approved-test-paths>

禁止来源：
- <draft-doc-paths>
- <historical-review-paths>
- <deleted-or-superseded-paths>

必须包含：
- 节点：
  - ...
- 边：
  - ...

禁止包含：
- ...

输出要求：
1. 先给 Mermaid 源码。
2. 再列出“节点依据”和“边依据”。
3. 最后列出“不确定项 / 待确认项”。
```

## 使用要求

- 生成前先读目标仓库自己的文档规则和架构图约束。
- 输入中的路径使用仓库根相对路径。
- 如果图表会进入仓库，优先把源码落到 owning 目录，避免只保留图片。
