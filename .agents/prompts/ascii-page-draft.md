# ASCII 页面草稿提示词

用于生成类似架构文档中 `页面草稿` 的简约 ASCII 线框图。

```text
请为指定页面生成简约 ASCII 页面草稿。

要求：
- 只输出页面草稿，不输出解释。
- 使用 ```text 代码块包裹。
- 使用 box drawing 字符：┌ ┐ └ ┘ ├ ┤ ─ │。
- 草稿宽度控制在 44 到 60 个字符之间。
- 每一行必须等宽，边框对齐。
- 只表达页面结构和主要区域，不写实现细节。
- 区域文案使用短英文或短中文标签，避免长句。
- 每个区域最多 1 到 3 行。
- 优先展示信息层级：顶部导航、筛选/工具栏、主体内容、操作区、状态区。
- 不要使用 emoji，不要使用 Markdown 表格，不要写说明段落。
- 如果页面有 loading、empty、error、degraded 状态，可以在草稿中用一行状态占位表达。
- 如果页面包含按钮，用简短标签表达，例如 [Save]、[Publish]、[Retry]。
- 如果页面包含列表或表格，用 2 到 3 行示例即可，不要展开太多数据。

页面名称：
{填写页面名称}

页面目标：
{填写用户在这个页面要完成的任务}

必须包含的区域：
{列出必须出现的区域}

状态要求：
{正常态 / 加载态 / 空态 / 错误态 / degraded 态}
```
